# pybracket Studio

An in-browser studio for the [pybracket](https://github.com/ProjectRio/pybracket) tournament
library. Pick a format, generate a bracket, click through results, and watch exactly how the
library builds and advances it — including the live `Bracket` JSON.

The twist: there is **no backend**. The studio runs the *real* pybracket Python library in your
browser via [Pyodide](https://pyodide.org/) (CPython compiled to WebAssembly). pybracket has zero
runtime dependencies, so it ships as a tiny pure-Python wheel that loads with `micropip`. Every
bracket you see was generated by the same code the test suite exercises — not a re-implementation.

## Stack

Modeled on the Project Rio frontend's Next.js migration:

- **Next.js 15** (App Router) + **React 19** + **TypeScript** (strict)
- **Tailwind CSS v4** (CSS-first `@theme`)
- **Zod** (available for input schemas)
- **Pyodide** (loaded from the jsDelivr CDN at runtime)
- Deploys to **Cloudflare Workers** via `@opennextjs/cloudflare`

## Getting started

```bash
npm install
npm run dev
```

Open http://localhost:3000. On first load the page downloads the Pyodide runtime and installs the
pybracket wheel (a one-time, few-second step shown in the loader). After that, generation and result
reporting are instant and fully offline.

## How it works

```
BuilderForm ──> lib/spec.ts ──> { op, ... } action
                                     │
            lib/pyodide.ts (Pyodide + micropip install of the wheel)
                                     │
                       public/py/bridge.py  dispatch(action_json)
                                     │
                        the real pybracket library
                                     │
        { ok, bracket, query, signals? } ──> BracketCanvas / Standings / JSON
```

- `lib/pyodide.ts` — loads Pyodide once, installs `public/pybracket-*.whl`, runs the bridge, and
  exposes a single synchronous `dispatch(action)`.
- `public/py/bridge.py` — a thin wrapper over pybracket's public API. Each action returns a JSON
  envelope with the serialized bracket plus pre-computed read models (ready matches, standings,
  placements, winner, completion) so the UI never re-derives library logic.
- `components/` — the studio UI: builder, interactive bracket canvas, standings/placements, and a
  live JSON inspector.

## Keeping the library in sync

The wheel in `public/` is built from the sibling `../pybracket` source. After changing the library,
rebuild and copy it in:

```bash
npm run sync-pybracket
```

(Override the source location or interpreter with `PYBRACKET_DIR` / `PYBRACKET_PY`.)

## Supported formats

Single elimination, double elimination, round robin, Swiss (Dutch/Monrad), and gauntlet
(single + dual, with optional opponent choice) are fully interactive. Each supports auto-play
(seeded or random), single-match reporting, result unwinding, and — for Swiss — round advancement.

**Not yet wired up:** the composite *pools → bracket* format (`generate_pools`), which is a
multi-stage flow (play the pool round-robins, then reseed into an elimination bracket). The bridge
in `public/py/bridge.py` is the place to add it.

## Deploying to Cloudflare

```bash
npm run preview   # build + local Workers preview

```
